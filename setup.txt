# Finance Backend API Setup Guide

This guide explains how to set up and run the Finance Backend API project locally.

## 1. Prerequisites

Before you begin, ensure you have the following installed:
- **Python 3.8 or higher**: Download from [python.org](https://www.python.org/downloads/).
- **Git**: For cloning the repository. Download from [git-scm.com](https://git-scm.com/downloads).
- **pip**: Python package installer (usually comes with Python).

You can verify installations by running:
```bash
python --version
git --version
pip --version
```

## 2. Clone the Repository

Clone the project and open the folder:

```bash
git clone https://github.com/Parbeen27/FastApi_Backend
cd FastApi_Backend
```

## 3. Configure Environment Variables

Create a local `.env` file from the example:

```bash
cp .env.example .env
```

Edit `.env` if you need to update project-specific settings.

## 4. Create a Virtual Environment

Use a virtual environment to isolate project dependencies.

```bash
python -m venv venv
```

### Activate the Virtual Environment

- On macOS/Linux:
  ```bash
  source venv/bin/activate
  ```
- On Windows:
  ```bash
  venv\Scripts\activate
  ```

When the environment is active, your prompt should show `(venv)`.

## 5. Install Dependencies

Install the required Python packages from `requirements.txt`:

```bash
pip install -r requirements.txt
```

If you use `pip-tools`, you can also run:

```bash
pip-sync
```

## 6. Seed Example Users (Optional)

If the project provides a seed script, run it to create test users:

```bash
python seed_user.py
```

This step is optional but useful for testing.

## 7. Run the Server

Start the FastAPI server with Uvicorn:

```bash
uvicorn main:app --reload
```

- `--reload` reloads the server automatically when code changes.
- Open the API at `http://127.0.0.1:8000`.

## 8. Access API Documentation

Once the server is running, open your browser and go to:

http://127.0.0.1:8000/docs

This page shows the interactive API documentation and allows you to test endpoints.

## 9. Troubleshooting

- If port `8000` is already in use:
  ```bash
  uvicorn main:app --reload --port 8001
  ```
- If you get permission errors on Windows, run your terminal as Administrator.
- If the virtual environment is not active, deactivate and reactivate it:
  ```bash
deactivate
source venv/bin/activate
```
- If dependency installation fails, upgrade pip:
  ```bash
pip install --upgrade pip
```
- If the server does not start, check `main.py` for syntax issues and make sure dependencies are installed.

If you still have trouble, refer to the project `README.md` for more details.
